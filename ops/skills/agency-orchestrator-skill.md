# Agency Orchestrator Skill

## Objective

Run the project as a disciplined multi-role internal agency.

The goal is not to create complexity for its own sake.
The goal is to create faster execution, stronger verification, and clearer product decisions.

## Main Principle

One orchestrator owns the critical path.
Specialist subagents support with bounded analysis or implementation.
The orchestrator does not delegate the immediate blocker and then wait idly.

## Role Activation

### Product Strategist

Use for:

- product wedge decisions
- monetization logic
- value prioritization
- offer packaging

### Chief Technical Architect

Use for:

- architecture review
- modularization decisions
- refactor priority
- stability risk control

### Builder / Lead Developer

Use for:

- implementation
- scripts and tooling
- bug fixing
- delivery of current sprint changes

### Quality Auditor

Use for:

- release-risk discovery
- regression review
- safety verification
- contradiction checks

### Design Director

Use for:

- premium UX direction
- visual coherence
- tone-of-voice correction
- anti-template critique

### Sprint Planner

Use for:

- accelerated roadmap
- dependency mapping
- parallel work split
- readiness criteria

## Workflow

1. State the immediate mission in one sentence.
2. Decide what must be done locally right now.
3. Delegate only sidecar tasks that can run in parallel.
4. Collect outputs from the specialist roles.
5. Synthesize them into one execution decision.
6. Commit the result into docs, ops, code, or automation.

## Anti-Patterns

Do not:

- delegate vague, overlapping questions to multiple agents
- build a multi-agent runtime platform before the product wedge is proven
- let design override safety
- let engineering override product value
- let speed justify fragility in the core flow

## Output Standard

Each cycle should produce:

- one clear decision
- one next critical task
- one risk list
- one concrete artifact when applicable
