# Biomarker Extraction Decision — 2026-05-06

Статус: принято частично
Источник: предложение Гипсика по расширению маркеров и DeepSeek-извлечению

## 1. Короткий вердикт

Предложение полезное, но формулировка “гарантирует 100% показателей” отклонена как небезопасная.

Решение:

- принять безопасную часть: расширить deterministic marker aliases для стандартных лабораторных и нутрициологически релевантных групп;
- не внедрять DeepSeek extraction как источник фактов без confidence/merge/audit;
- не добавлять высокорисковые и небиомаркерные категории в общий авто-парсер без отдельной safety-маршрутизации.

## 2. Что сделано Codex

В `WellnessBot/lab_ocr.py`:

- расширен `KNOWN_MARKER_ALIASES` для стандартных лабораторных групп;
- добавлены железо/ОАК, витамины, щитовидная панель, углеводный и липидный обмен, печень/ЖКТ, почки/электролиты, воспаление, часть гормонов, минералы;
- исправлена поддержка десятичных значений с точкой, например `12.4`, `5.1`, `9.0-30.4`.

## 3. Что намеренно НЕ добавлено

Не добавлено в общий авто-парсер:

- онкомаркеры;
- инфекции/ПЦР;
- УЗИ/рентген/визуальные обследования;
- широкие аллергопанели;
- любые группы, где автоматическое попадание в клиентское досье может создать медицински рискованную трактовку.

Причина: распознать название показателя технически можно, но интерпретировать такие данные в нутрициологическом продукте без отдельного маршрута к врачу небезопасно.

## 4. DeepSeek extraction — решение

Draft `HERMES-20260505-027_DEEPSEEK_BIOMARKER_EXTRACTION_DRAFT.md` пока не внедряется.

Причины:

- AI не может гарантировать 100% безошибочное извлечение;
- при плохом OCR AI может “дорисовать” структуру;
- это не решает текущий live-риск `Yandex OCR 401 Unauthorized`;
- нужен merge/audit протокол: deterministic факты имеют приоритет, AI может только предложить кандидаты.

Безопасный будущий вариант:

1. deterministic parsing first;
2. AI candidate extraction second;
3. no invented values;
4. confidence/source_line required;
5. high-risk markers require doctor-route flag;
6. uncertain candidates не используются как факты;
7. client/operator видит “нужно подтвердить показатель”, если confidence низкий.

## 5. Проверки

Проверено:

- `py_compile WellnessBot/lab_ocr.py`;
- новые маркеры распознаны: сывороточное железо, трансферрин, ОЖСС, ГГТ, ЩФ, липаза, гомоцистеин, цинк, селен, АМГ;
- базовые маркеры не сломались: ферритин, витамин D, ТТГ, глюкоза.

## 6. Следующий правильный шаг

Не расширять список бесконечно, а закрыть P0 file/lab flow:

1. Проверить причину Yandex OCR `401 Unauthorized` без вывода ключей.
2. Убедиться, что при OCR error клиент получает понятный fallback.
3. Добавить live-test cases: PDF, хорошее фото, плохое фото, ручной ввод.
4. После этого отдельно проектировать AI-assisted extraction как P2/P3 улучшение.
