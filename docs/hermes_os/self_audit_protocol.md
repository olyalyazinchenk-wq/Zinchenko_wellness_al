# Self-Audit Protocol

## После каждого значимого действия — 8 вопросов

1. **Scope:** не нарушил ли я разрешённый scope?
2. **Secrets:** не затронул ли я `.env`, ключи, токены, пароли?
3. **Client data:** не прочитал/скопировал ли я реальные клиентские данные?
4. **Evidence:** не сделал ли вывод без доказательств?
5. **Bloat:** не раздул ли задачу? можно ли упростить результат?
6. **Verification:** есть ли проверка результата (минимальная, но реальная)?
7. **Next step:** понятен ли следующий шаг (1 конкретное действие)?
8. **Extra files:** не создал ли я лишний файл/артефакт?

## Формат в отчёте

```text
### Self-audit
- Scope: ✅ / ❌
- Secrets: ✅
- Client data: ✅
- Evidence: ✅
- Bloat: ✅ (N файлов, все нужны)
- Verification: ✅ (перечислены проверенные команды/файлы)
- Next step: [конкретно]
- Extra files: 0
```

## Red flags — немедленная остановка

- Открыл/прочитал `.env` → STOP: не сохранять, не цитировать, не коммитить.
- Увидел клиентский PDF/фото/анализы → STOP: закрыть, не анализировать, не копировать.
- Потребовалось менять платежи/токены/провайдеров → STOP: фиксировать вопрос и ждать человеческого решения/approval.
