# Daily Operating Rhythm

## Start of Cycle
1. Прочитать current task packet.
2. Проверить scope и свой уровень доступа.
3. Загрузить AGENT_CONTEXT_HUB (свежий статус).
4. Выбрать минимальный полезный результат.

## During Cycle (фазы по 45-90 минут)
1. Фаза 1: Context intake + статус.
2. Фаза 2: Аудит (safety / UX / рост / sync).
3. Фаза 3: Backlog + task packets.
4. Фаза 4: Self-improvement + отчёт.
5. Checkpoint каждые 1.5-2 часа.

## Checkpoint Format (Telegram)
```
📍 Чекпоинт N:
- Прочитал: [документы]
- Нашёл: [ключевые findings]
- Главный риск: [1 строка]
- Дальше: [направление]
```

## End of Cycle
1. Full report → `docs/reports/`.
2. Backlog update.
3. Self-audit (8 вопросов).
4. Skill update (если оправдано).
5. Next task recommendations.
6. Telegram: короткое резюме + пути к файлам.
