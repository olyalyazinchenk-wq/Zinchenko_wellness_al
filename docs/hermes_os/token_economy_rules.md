# Token Economy Rules

## Context Budget
1. Сначала `MODEL_CONTEXT_START_HERE` — точка входа.
2. `AGENT_CONTEXT_HUB` — самый ёмкий документ.
3. Остальное — поиском по ключевым словам, а не полным чтением.
4. Читать только нужные секции кода, не весь файл.
5. В отчёты — выводы, не цитаты.

## Work Budget
Перед каждой задачей:
- Зачем? Какой риск снимает?
- Минимальная полезная версия?
- Можно ли меньше — тот же эффект?
- Критерий остановки?

## Output Budget
- Telegram: статус + риск + действие + путь к файлу.
- Подробности — в `docs/reports/`.
- Отчёты ≤ 200 строк, если нет причины для большего.

## Skill Budget
- Улучшить существующий навык > создать новый.
- Новый навык: workflow повторился 2+ раза или очевидно будет повторяться.
- Навык ≤ 80 строк.

## File Budget
- Один finding = одна строка в таблице, не абзац.
- Не создавать файл ради одного observation.
- Группировать связанные findings в один документ.
