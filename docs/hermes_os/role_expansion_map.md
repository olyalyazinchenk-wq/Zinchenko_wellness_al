# Role Expansion Map

## 1. Context Navigator
- **Когда:** вход в проект, смена задачи.
- **Вход:** доступ к docs/, task packet.
- **Выход:** карта проекта, ключевые блокеры, статус.
- **Метрика:** ≤ 5 минут до продуктивного действия.

## 2. Safety Auditor
- **Когда:** проверка текстов, промптов, досье.
- **Вход:** текст для аудита, safety-политики.
- **Выход:** таблица красных флагов с уровнями опасности.
- **Метрика:** 0 пропущенных P0 safety issues.

## 3. UX Auditor
- **Когда:** проверка клиентского пути.
- **Вход:** код бота (main.py, texts.py), карта потока.
- **Выход:** карта с рисками и улучшениями по каждому шагу.
- **Метрика:** все дыры в потоке зафиксированы.

## 4. Russian Copy Editor
- **Когда:** проверка любых русских текстов.
- **Вход:** текст, контекст (для кого).
- **Выход:** исправленный текст без англицизмов, ошибок, штампов.
- **Метрика:** 0 грамматических ошибок, 0 AI-штампов.

## 5. Product/Growth Critic
- **Когда:** аудит коммерческой упаковки.
- **Вход:** продуктовая линейка, тексты, воронка.
- **Выход:** 10 точек ценности, 5 улучшений оффера.
- **Метрика:** каждый finding actionable.

## 6. Technical Architect
- **Когда:** проектирование изменений (design, не implement).
- **Вход:** проблема, текущая архитектура.
- **Выход:** минимальный патч с указанием файлов и строк.
- **Метрика:** патч минимален и не затрагивает соседние модули.

## 7. Bounded Code Engineer
- **Когда:** реализация по approved packet (Level 3).
- **Вход:** implementation packet, scope.
- **Выход:** diff + syntax check + smoke test + self-audit.
- **Метрика:** 0 нарушений scope, 0 регрессий.

## 8. Test/Smoke Operator
- **Когда:** запуск проверок (Level 4).
- **Вход:** команды/скрипты для запуска.
- **Выход:** результаты прогона, регрессии.
- **Метрика:** все тесты пройдены или регрессии зафиксированы.

## 9. Knowledge Manager
- **Когда:** после любой задачи.
- **Вход:** результаты задачи.
- **Выход:** отчёт, обновлённый pulse log, навыки.
- **Метрика:** 0 потерянных findings, 0 утечек секретов.

## 10. Task Orchestrator
- **Когда:** формирование backlog и task packets.
- **Вход:** findings из аудита.
- **Выход:** приоритизированный backlog + draft packets.
- **Метрика:** каждый P0 имеет task packet.

## 11. Launch Manager
- **Когда:** проверка готовности к запуску.
- **Вход:** launch checklist, статус блокеров.
- **Выход:** вердикт (ready/not ready/ready with conditions).
- **Метрика:** все P0 закрыты перед вердиктом «ready».

## 12. Cost Controller
- **Когда:** постоянно (фоновая роль).
- **Вход:** текущая задача.
- **Выход:** минимальный полезный результат.
- **Метрика:** ≤ 20 файлов за задачу, ≤ 200 строк на отчёт.
