### Definition of done

A task is not done unless:

- it reduces ambiguity around the active same-user path
- it improves runtime-storage coherence
- it improves routed-vs-model truth for live dialogue
- it preserves legal and safety boundaries
- it reduces environment failure risk when disk headroom is low
- it survives available checks, logs, benchmark evidence, and operator review
- it makes the next paid cycle easier to explain and deliver
- it removes a loop instead of creating another

- `C:` remains below the safety floor and non-critical generation still continues
- more than one active same-user path exists across runtime and persisted state

- unreadable or unconfirmed labs are being treated as facts
- `20260425T214914Z_<REDACTED_ID>` is treated as delivery-ready while `requires_lab_resubmission=true`

- public launch language gets ahead of legal, payment, support, or delivery readiness

## Loop-Break Rule

If disk headroom collapses, a same-user path conflict appears, or the benchmark remains fully routed:

1. stop
2. restore headroom above the safety floor
3. classify each case item as runtime intake, persisted case, evidence-only, or archive
4. classify each reply path as deterministic safety/logistics or model-owned reasoning
5. keep exactly one active paid path
6. freeze or archive the rest
7. rerun the benchmark before any new prompt, pricing, growth, or UI discussion continues

No silent same-user path multiplication, silent template capture, or silent environment drift is allowed.
