# Telegram-бот: Нутрициологический магазин + AI-разбор

> **Архитектурный план проекта**

---

## 🔧 Технический стек (принятые решения)

| Компонент | Технология | Почему |
|-----------|-----------|--------|
| **Язык** | Python 3.11+ | Широчайший зоопарк библиотек |
| **Фреймворк бота** | aiogram 3.x | Асинхронный, FSM, платежи, огромное комьюнити |
| **База данных** | PostgreSQL + SQLAlchemy async / Tortoise-ORM | Надёжно, миграции (Alembic) |
| **Платежи** | ЮMoney/ЮKassa через Telegram Payments API | Нативный платёжный API Telegram |
| **AI-разбор** | DeepSeek (или GPT-4o) через API | Умный анализ анкет и анализов |
| **OCR** | Tesseract + EasyOCR | Распознавание фото анализов |
| **Файлы** | MinIO / локальная файловая система | Хранение загруженных анализов |
| **Админка** | Сначала в боте (скрытые команды), потом Telegram WebApp | Быстрый старт, потом прокачка |
| **Биллинг** | Таблицы payment + invoice в БД | Полная история транзакций |

```

2. Главное меню:

   📋 Мои анкеты
   📊 Мои разборы
   ❓ Помощь

3. 🛒 Выбор услуги:
   → Магазин: список категорий
     • 🩸 Питание при диабете
     • ❤️ Питание при гипертонии
     • 🦋 Питание при щитовидной железе
     • ⚖️ Питание при лишнем весе
     • 🩺 Питание при анемии
     • 🫃 Питание при ЖКТ
   
   → Выбор конкретной услуги внутри категории:
     • «Первичная консультация» — 2 500 ₽

     • «Расширенный разбор» — 5 000 ₽
       (всё выше + план питания + неделя сопровождения)

4. 💳 Оплата:
   → Telegram Stars / ЮKassa через Invoice
   → После оплаты → статус PAID
   → Бот присылает ссылку на анкету

5. 📋 Анкетирование (FSM — пошагово):
   Бот задаёт вопросы последовательно:
   
   ⏺️ Базовый блок (для всех направлений):
     1. Ваши ФИО
     2. Возраст, пол, рост, вес
     3. Основная жалоба/запрос
     4. Цель обращения
     5. Режим питания (сколько раз в день, какие продукты)
     6. Примерный рацион за день
     7. Питьевой режим
     8. Физическая активность
     9. Сон (часы, качество)
     10. Уровень стресса
     11. Принимаете ли БАДы/лекарства
     12. Хронические заболевания
     13. Аллергии/непереносимости

     (вопросы из category_specific_fields по slug направления)

     «Загрузите фото или PDF ваших анализов (или пропустите)»

     → Можно ввести показатели вручную

   4. Сохраняется в ai_reviews как DRAFT

     **Дефициты:**
     [выявленные дефициты]
     
     **Образ жизни:**
     [рекомендации]
     
     **Рекомендуемые обследования:**
     [список анализов]
     
     **План действий:**
```

## 🧠 AI-разбор: как это работает (детали)

### Промпт-система

Базовый промпт задаёт роль и правила:

```
Ты — ассистент нутрициолога. Твоя задача — проанализировать анкету клиента
и лабораторные анализы (если предоставлены) и дать предварительный
нутрициологический разбор.

ВАЖНЫЕ ПРАВИЛА:
- Ты НЕ ставишь медицинские диагнозы
- Ты НЕ назначаешь лечение
- Твои рекомендации — это нутрициологические наблюдения: питание, 
  образ жизни, дефициты, поддержка организма, профилактика
- Всегда указывай: «Данные рекомендации основаны на анкете и анализах,
  но не заменяют очную консультацию специалиста»
- Будь тактичной и поддерживающей
- Конкретные рекомендации по питанию (а не общие фразы)
```

+ **Специфический промпт** для каждого направления:
  - для диабета — про ГИ, гликемическую нагрузку, углеводы
  - для гипертонии — про натрий, калий, магний
  - и т.д.

### Формат вывода AI (структурированный JSON)

```json
{
  "nutrition_recommendations": "...",
  "lifestyle_recommendations": "...",
  "deficiencies_found": "...",
  "additional_tests": "...",
  "action_plan": "...",
  "risk_factors": ["...", "..."],
  "urgency_level": "normal|attention|urgent"
}
```

### OCR-пайплайн для анализов

```
Фото / PDF клиента
    ↓
Конвертация в изображение (pdf2image)
    ↓
EasyOCR / Tesseract → распознанный текст
    ↓
DeepSeek: «Извлеки лабораторные показатели из текста,
форматируй как JSON: {lab_name: value, unit: "..."}»
    ↓
Структурированные данные → в analysis_results_json
    ↓
Подаются в промпт AI-разбора вместе с анкетой
```

## 💳 Платежи: архитектура (ЮKassa + Telegram Payments)

```
1. Пользователь нажимает «Купить услугу»
2. Сервер создаёт Invoice (счёт) через Telegram API:
   - Название: услуга
   - Цена: в рублях
   - Provider Token: ЮKassa
3. Telegram открывает платёжный интерфейс
4. Пользователь оплачивает через ЮKassa
5. Telegram присылает pre_checkout_query → ответ OK
6. Telegram присылает successful_payment → сервер фиксирует
7. Статус заказа → PAID
8. Бот отправляет ссылку на анкету
```

**Безопасность:**
- Все платежи через Telegram Payments API — бот не хранит платёжные данные
- Provider Token в .env, не в коде
- Логирование всех платежей в БД (id, сумма, статус, дата)

## 🚀 Этапы разработки (дорожная карта)

### 🟢 Этап 1: База (MVP — 2 недели)

**Цель:** Работающий магазин + оплата + анкета

1. **Каркас проекта**
   - Инициализация проекта, requirements.txt
   - config.py с переменными окружения
   - Подключение БД (PostgreSQL + SQLAlchemy/Tortoise)
   - /start с приветствием
   - Базовые модели (users, services, orders)

2. **Магазин услуг**
   - Таблицы service_categories + services
   - Хендлер shop.py: показать категории → услуги
   - Inline-клавиатуры

3. **Платежи**
   - Интеграция Telegram Payments API + ЮKassa
   - Обработка pre_checkout_query
   - Обработка successful_payment
   - Статусы заказов

4. **Анкетирование**
   - FSM: базовый блок (12 вопросов)
   - Сохранение в БД
   - Статусы анкет

5. **Админ-уведомления**
   - При новой анкете → уведомление админу в бота
   - Статусы заказов

**Результат:** Клиент может выбрать услугу → оплатить → заполнить анкету

### 🟡 Этап 2: AI-разбор (1-2 недели)

1. **Загрузка анализов**
   - Multi-step: фото/PDF/ручной ввод
   - Хранение файлов
   - Легкое распознавание (OCR)

2. **AI-интеграция**
   - Подключение DeepSeek API
   - Система промптов (базовые + по направлениям)
   - Генерация разбора
   - Сохранение структурированного результата

3. **Выдача результата**
   - Клиент получает уведомление
   - Красивый вывод разбора (с эмодзи, разделами)
   - Возможность получить PDF-версию разбора
   
4. **Админ-проверка разбора**
   - Нутрициолог может просмотреть AI-разбор
   - Подтвердить или отредактировать
   - Опубликовать клиенту

**Результат:** Полный цикл: оплата → анкета → анализы → AI-разбор → выдача

### 🔵 Этап 3: Прокачка (по желанию)

1. **Telegram WebApp** для админки
   - Управление услугами (добавить/редактировать)

## 🎨 Как добавить новую услугу (расширяемость)

Система спроектирована так, что для добавления нового направления нужно:

1. **Создать категорию** (через админку или SQL):
   - INSERT INTO service_categories (name, slug, description, icon)
   - Пример: slug = `kidney_disease`

2. **Добавить услуги** в категории:
   - INSERT INTO services (category_id, name, price_rub, ...)

3. **Добавить специфические вопросы** в систему:
   - Через таблицу question_templates (2-я очередь)
   - Или через файл конфигурации questionnaires_config.yaml

4. **(Опционально) Написать промпт** для AI:
   - ai_prompts/kidney_disease.txt
   - Специфические знания по питанию при болезнях почек

5. **Готово!** Всё остальное — общая логика — уже работает
