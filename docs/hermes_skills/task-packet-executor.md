# Task Packet Executor

## Когда использовать
При получении утверждённой задачи (task packet со статусом `approved`).

## Входные данные
- Task packet из `docs/tasks/HERMES-*.md`.
- Контекст проекта (см. project-context-loader).

## Пошаговый workflow

1. Прочитать task packet полностью: Objective, Allowed Scope, Hard Rules, Deliverables, Acceptance Criteria.
2. Составить план: маленькие проверяемые шаги.
3. Выполнять шаг за шагом, фиксируя изменения.
4. После каждого шага проверять: не нарушены ли запреты.
5. Собрать deliverables.
6. Проверить acceptance criteria.
7. Написать отчёт в `docs/reports/HERMES-YYYYMMDD-NNN_*.md`.

## Критерии качества
- Все шаги выполнены без нарушения границ.
- Отчёт содержит: что сделано, изменённые файлы, как проверено, риски.
- Нет утечки секретов или персональных данных.

## Что запрещено
- Менять код бота, `.env`, production-конфиги без отдельного разрешения.
- Менять оплату, клиентскую выдачу, VPS.
- Трогать клиентские данные без явного разрешения в task packet.
- Работать без статуса `approved`.
- Если задача требует правки кода — создать draft task packet для Codex, а не править самому (Level 1).

## Формат результата
Отчёт: 1) Что понял, 2) Что сделал, 3) Изменённые файлы, 4) Как проверил, 5) Риски/блокеры, 6) Следующий шаг.
