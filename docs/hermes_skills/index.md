# Hermes Skills Index

Дата создания: 2026-05-05
Назначение: индекс рабочих навыков Гермеса для проекта Olga Wellness AI.
Правило: каждый навык — короткий, прикладной, с чёткими границами.

## Навыки

| # | Навык | Файл | Назначение |
|---|-------|------|------------|
| 1 | Project Context Loader | `project-context-loader.md` | Быстрый вход в контекст проекта без лишних вопросов |
| 2 | Task Packet Executor | `task-packet-executor.md` | Взять утверждённую задачу и довести до отчёта |
| 3 | Wellness Safety Auditor | `wellness-safety-auditor.md` | Проверка медицинско-юридических рисков в текстах |
| 4 | Premium Russian Copy Editor | `premium-russian-copy-editor.md` | Убрать шаблонность, англицизмы и ошибки из русских текстов |
| 5 | Telegram Bot Flow Auditor | `telegram-bot-flow-auditor.md` | Проверка клиентского пути в Telegram-боте |
| 6 | Growth and Offer Critic | `growth-and-offer-critic.md` | Проверка коммерческой ценности и упаковки продукта |
| 7 | Knowledge Sync Reporter | `knowledge-sync-reporter.md` | Фиксация результатов без утечки секретов |
| 8 | Launch Readiness Checker | `launch-readiness-checker.md` | Проверка готовности к pilot/public launch |
| 9 | Lab Data Safety Reviewer | `lab-data-safety-reviewer.md` | Проверка безопасности работы с лабораторными данными |
| 10 | Agent Self-Improvement Loop | `agent-self-improvement-loop.md` | Цикл самоулучшения после каждой задачи |

## Как использовать

1. При входе в проект: #1 (Project Context Loader).
2. При получении задачи: #2 (Task Packet Executor).
3. При проверке текстов: #4 (Premium Russian Copy Editor) + #3 (Wellness Safety Auditor).
4. При аудите бота: #5 (Telegram Bot Flow Auditor).
5. При проверке продукта: #6 (Growth and Offer Critic).
6. При работе с анализами: #9 (Lab Data Safety Reviewer).
7. При подготовке к запуску: #8 (Launch Readiness Checker).
8. После любой задачи: #10 (Agent Self-Improvement Loop) + #7 (Knowledge Sync Reporter).

## Универсальные навыки (переносимые)

Навыки #1, #2, #7, #10 являются универсальными и могут использоваться в других проектах после адаптации.
Навыки #3, #4, #5, #6, #8, #9 содержат проектную специфику и требуют пересмотра перед переносом.

## Правило обновления

После каждой задачи проверять: нужно ли создать новый навык или обновить существующий. 
Результат фиксировать в отчёте задачи.
