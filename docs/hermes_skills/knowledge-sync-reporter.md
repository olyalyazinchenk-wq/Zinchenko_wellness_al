# Knowledge Sync Reporter

## Когда использовать
После выполнения любой задачи — для фиксации результатов в правильных каналах без утечки секретов.

## Входные данные
- Результаты выполненной задачи.
- Изменённые файлы.
- Остаточные риски.

## Пошаговый workflow

1. **Отчёт в `docs/reports/`:**
   - Формат: `HERMES-YYYYMMDD-NNN_краткое_описание.md`.
   - Содержит: что сделано, изменённые файлы, как проверено, риски.
   - Не содержит: токенов, паролей, клиентских данных, Telegram ID.

2. **Обновление pulse log (если разрешено):**
   - Добавить запись в `docs/PROJECT_PULSE_LOG.md`.
   - Формат: дата, время MSK, краткое описание, статус блокеров.

3. **GitHub (если настроен доступ):**
   - Проверить, что в коммите нет запрещённых данных.
   - Коммитить только разрешённые файлы (`docs/`, `ops/`).
   - Не коммитить: `.env`, `WellnessBot/data/`, клиентские файлы.

4. **Notion (если настроен доступ):**
   - Обновить статусы задач.
   - Не переносить в Notion персональные данные и секреты.

5. **Obsidian (если используется):**
   - Зеркалировать в `docs/obsidian_mirror/` без клиентских данных.

## Критерии качества
- Отчёт создан в правильной папке.
- Нет утечки секретов или персональных данных.
- Все изменённые файлы перечислены.

## Что запрещено
- Экспортировать реальные клиентские данные в GitHub/Notion.
- Сохранять токены, ключи, пароли в отчётах.
- Коммитить `WellnessBot/data/`.

## Формат результата
Перечень созданных/обновлённых артефактов с абсолютными путями.
