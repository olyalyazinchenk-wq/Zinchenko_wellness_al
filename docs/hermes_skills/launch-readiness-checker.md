# Launch Readiness Checker

## Когда использовать
При подготовке к pilot launch или public launch, или при проверке текущего состояния перед важным шагом.

## Входные данные
- Критерии из `docs/PROJECT_DEVELOPMENT_SINGLE_SOURCE_OF_TRUTH.md` (разделы 16-17).
- Текущий статус из `docs/AGENT_CONTEXT_HUB.md`.
- Результаты QA (`ops/reports/quality_report_*.md`).

## Пошаговый workflow

1. **Pilot readiness (controlled concierge):**
   - [ ] Бот стабильно запущен и evidenced как running (ПРОВЕРИТЬ ФАКТИЧЕСКИ, не по документам).
   - [ ] Delivery gate: `delivered_to_client` блокируется при `needs_revision`/`must_rewrite` без manual override.
   - [ ] Multi-path drift устранён: 1 канонический путь на пользователя.
   - [ ] Mini-app не показывает неверные цены (2990₽) и medical findings.
   - [ ] Интерфейс только на русском (включая CTA и термины).
   - [ ] Продуктовая витрина понятна.
   - [ ] Демо-пример показывается.
   - [ ] Анкета проходит без зависания.
   - [ ] Файлы принимаются.
   - [ ] При плохом качестве — просит переслать.
   - [ ] Ручная оплата работает организационно.
   - [ ] Админ видит продукт, сумму, заявку.
   - [ ] AI-черновик не уходит клиенту без human review.
   - [ ] Нет диагнозов и лечения.
   - [ ] Есть понятный план клиенту.
   - [ ] Invoice payload содержит правильный offer_code (не хардкод "premium:").
   - [ ] Manual override имеет audit trail (кто, когда, почему).

2. **Public launch readiness:**
   - [ ] Минимум 3-5 реальных клиентских циклов.
   - [ ] Нет критических сбоев в анкете, файлах, оплате, досье, админке.
   - [ ] Human review работает стабильно.
   - [ ] Понятен порядок хранения данных.
   - [ ] Юридически аккуратный способ оплаты.
   - [ ] Есть отзывы и подтверждённая ценность.
   - [ ] VPS/24-7 решение принято.

3. **Сформировать список невыполненных критериев.**

4. **Для каждого невыполненного критерия:**
   - Почему не выполнено.
   - Что нужно сделать.
   - Кто ответственный.

## Критерии качества
- Каждый критерий проверен по факту, а не «кажется».
- Невыполненные критерии приоритизированы.

## Что запрещено
- Закрывать критерии без реальной проверки.
- Игнорировать красные флаги ради «скорейшего запуска».

## Формат результата
Чек-лист с отметками [x]/[ ] и комментариями по каждому пункту.
