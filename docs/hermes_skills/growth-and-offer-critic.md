# Growth and Offer Critic

## Когда использовать
При проверке коммерческой ценности продукта, упаковки, позиционирования, воронки продаж.

## Входные данные
- Продуктовая линейка: `docs/PRODUCT_LINE_V2_20260426.md`.
- Тексты: меню, кнопки, демо-пример, лендинг (если есть).
- Правила безопасности: `docs/NUTRITION_NAVIGATION_POLICY_20260420.md`.

## Пошаговый workflow

1. **Понятность ценности:**
   - Понимает ли клиент за 15 секунд, что это за продукт?
   - Отличается ли от бесплатных советов из интернета?
   - Есть ли причина выбрать Ольгу, а не конкурента?

2. **Упаковка продукта:**
   - Демо-пример: показывает ли качество без раскрытия всего?
   - Карточки продуктов: понятны ли различия между 7/30/VIP?
   - Цены: соответствуют ли официальной линейке (3900 / 6900 / 14900)?

3. **Воронка:**
   - Есть ли понятный следующий шаг после каждого экрана?
   - Где клиент может «отвалиться» без действия?
   - Есть ли CTA (призыв к действию) на каждом ключевом экране?

4. **Доверие:**
   - Упомянута ли безопасность (не диагноз, не лечение)?
   - Есть ли юридические страницы (если сайт/лендинг)?
   - Есть ли социальное доказательство (отзывы, примеры)?

5. **Отстройка от шаблонности:**
   - Не выглядит ли продукт как «ещё один ChatGPT-совет»?
   - Есть ли конкретика, а не общие фразы?

## Критерии качества
- Выявлены слабые места воронки.
- Проверено соответствие цен.
- Даны конкретные рекомендации по усилению.

## Что запрещено
- Менять цены самовольно.
- Обещать результаты / доход без данных.
- Использовать манипулятивные приёмы.

## Формат результата
1. Оценка воронки (сильные / слабые места).
2. Таблица несоответствий.
3. 3-5 конкретных рекомендаций.
