# Project Context Loader

## Когда использовать
При первом подключении к проекту Olga Wellness AI, после долгой паузы, или при смене задачи.

## Входные данные
- Доступ к `docs/` проекта.
- ID задачи (если есть).

## Пошаговый workflow

1. Прочитать `docs/MODEL_CONTEXT_START_HERE_20260505.md` — точка входа.
2. Прочитать `docs/AGENT_CONTEXT_HUB.md` — **самый информативный документ**: свежий статус, регрессии, блокеры, must-not-change rules. Читать полностью.
3. Прочитать последние 100 строк `docs/PROJECT_PULSE_LOG.md` — последние решения.
4. Прочитать `docs/HERMES_PROJECT_WORKER_PROTOCOL_20260505.md` — свои права.
5. Если есть task packet — прочитать его; если нет — не менять проект самовольно.
6. Проверить `git status` (если доступно).
7. Проверить свои навыки в `docs/hermes_skills/index.md` — возможно, есть подходящий навык для задачи.

## Критерии качества
- Гермес не задаёт общих вопросов «что за проект?».
- Гермес понимает текущий этап (controlled concierge pilot), блокеры, продуктовую линейку.
- Гермес знает свои границы: что можно, что нельзя.

## Что запрещено
- Читать `.env`, клиентские данные, `WellnessBot/data/` без явного task packet.
- Начинать изменения без approved-задачи.
- Задавать вопросы, ответы на которые есть в прочитанных документах.

## Формат результата
Краткий вывод: «Контекст загружен. Текущий этап: [controlled concierge pilot]. Ключевые блокеры: [...]. Готов к задаче [ID].»
