# OCR/File Fallback Playbook

## Контекст
Yandex Vision OCR (`lab_ocr.py:279`) возвращал `401 Unauthorized` при обработке фото. Требуется fallback без раскрытия ключей.

## Что принимает бот
- PDF из лаборатории (lab_ocr.py:281 — сначала extract_text, потом OCR)
- Фото: JPEG, PNG (lab_ocr.py:296-297)
- Ручной текстовый ввод показателей

## Цепочка OCR

| Шаг | Файл/функция | Успех | Ошибка |
|-----|-------------|-------|--------|
| 1 | PDF → extract_text_from_pdf (lab_ocr.py:202) | Возвращает текст | None, идёт к шагу 2 |
| 2 | Фото/PDF → Yandex OCR (lab_ocr.py:279) | Возвращает текст | None, идёт к шагу 3 |
| 3 | Клиент → ручной ввод | Текст показателей | — |
| 4 | parse_biomarkers (lab_ocr.py:332) | Список biomarkers | Пустой список |
| 5 | enrichment (nutrition_reference_ranges) | biomarkers с ranges | Без ranges |

## Что видит клиент СЕЙЧАС (texts.py)

- **LABS_GUIDANCE_TEXT (стр. 84):** «Если фото окажется нечётким... я попрошу прислать файл заново»
- **Проблема:** нет явного fallback-сообщения при технической ошибке OCR. Клиент может не понять, почему файл «не читается».

## Готовые клиентские тексты (copy proposals, НЕ применять в код)

### 1. Файл плохого качества
> «Файл не удалось прочитать достаточно уверенно — изображение может быть нечётким или содержать блики. Пожалуйста, пришлите более качественное фото или PDF из лаборатории. Либо введите ключевые показатели текстом: название анализа, значение и единицу измерения.»

### 2. OCR не сработал технически
> «Сейчас технически не удалось автоматически распознать файл. Это не означает, что с файлом что-то не так — просто системе нужна помощь. Вы можете: (1) прислать тот же файл в PDF, (2) сделать фото при дневном свете без бликов, (3) ввести 3-5 самых важных показателей текстом.»

### 3. Распознаны не все показатели
> «Часть показателей распознана, но не все строки читаются уверенно. Я использую только то, что видно чётко. Остальное — пришлите, пожалуйста, текстом или более качественным файлом, чтобы не перепутать значения.»

### 4. Показатель/единица под вопросом
> «В строке [название] значение или единица измерения видны нечётко. Чтобы не ошибиться в интерпретации, уточните, пожалуйста: [показатель] = [значение] [единица]? Или пришлите этот фрагмент отдельно.»

### 5. Клиент прислал фото жалобы, а не анализ
> «Я вижу фото. Чтобы не строить предположения по изображению, опишите, пожалуйста: что именно беспокоит, как давно, где локализация, усиливается ли при движении/нагрузке. Врачебный осмотр здесь не заменяется — если нужно, я подскажу, к какому специалисту обратиться.»

## Запрещено (safety gate)

- ❌ Использовать неуверенно распознанные значения как факты
- ❌ Выдумывать единицы измерения
- ❌ Строить supplement-схемы на сомнительных данных
- ❌ Говорить «анализы в норме» без уверенного распознавания

## Smoke/live cases для проверки

1. PDF с читаемым текстом → biomarkers найдены
2. Хорошее фото анализа → biomarkers найдены
3. Плохое фото (размытое) → client fallback №1
4. Техническая ошибка OCR (401) → client fallback №2
5. Фото не анализа (жалоба) → client fallback №5
6. Ручной ввод показателей → biomarkers найдены
