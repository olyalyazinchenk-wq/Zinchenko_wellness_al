# Live Test Triage System

## Шаблон карточки live feedback

```
Скрин/описание: [что Ольга прислала]
Шаг пути: [start / витрина / анкета / оплата / файлы / досье / доставка]
Проблема: [одна фраза]
Priority: P0/P1/P2/P3
Area: UX / safety / payment / lab / runtime / copy / legal / growth
Блокирует пилот: да/нет
Fix owner: Codex / Hermes / Olga
Действие: [конкретный следующий шаг]
```

## Классификация

### Priority
- **P0:** блокирует пилот (delivery не работает, оплата не проходит, досье опасно)
- **P1:** критично для качества (текст непонятен, файл не принимается)
- **P2:** улучшает опыт (лишний шаг, длинный текст)
- **P3:** косметика, технический долг

### Area
- **UX:** клиентский путь, кнопки, навигация
- **Safety:** красные флаги, диагнозы, лечение
- **Payment:** оплата, подтверждение, суммы
- **Lab:** OCR, файлы, анализы, качество
- **Runtime:** бот не отвечает, таймауты, ошибки
- **Copy:** тексты, перевод, тон
- **Legal:** согласие, privacy, PD
- **Growth:** ценность, конверсия, оффер

## Кто что делает

| Находка | Codex | Hermes | Olga |
|---------|-------|--------|------|
| Код/логика | Фиксит | Готовит draft packet | Подтверждает |
| Текст/копия | Применяет | Готовит copy proposal | Утверждает |
| Процесс/регламент | — | Готовит playbook | Выполняет |
| Юридическое | — | Готовит анализ | Решает |


## Критерии блокировки пилота

**Блокирует пилот, если:**
- Клиент МОЖЕТ получить досье без human review
- Клиент МОЖЕТ получить досье без оплаты
- Клиент видит диагноз или назначение лечения
- Оплата не работает
- Файлы не принимаются и нет fallback

**НЕ блокирует пилот, если:**
- Текст можно улучшить (но он безопасен)
- UX можно упростить (но путь проходим)
- Дизайн не идеален
