# Экономический расчет инфраструктуры бота

Дата расчета: 2026-04-24.

## Короткий вывод

Лучшее решение для запуска пилота: гибридная схема.

- Telegram-бот остается в одном чате: клиент никуда не уходит из Telegram, кроме внешних ссылок на оплату или сдачу анализов.
- DeepSeek используется для текстовых ответов, разборов, гипотез и PDF-структуры.
- Yandex SpeechKit используется только для распознавания голосовых сообщений.
- Yandex Cloud можно использовать как хостинг/инфраструктуру, если нужен стабильный запуск без домашнего компьютера.

Почему так: DeepSeek для текстовой генерации существенно дешевле YandexGPT Pro, а Яндекс SpeechKit удобен и недорог для русской речи.

## Маршрут клиента

### Если анализов нет

1. Клиент пишет боту в Telegram.
2. Бот собирает первичную анкету и жалобы.
3. Бот мягко предлагает сдать анализы через партнерскую ссылку HelloDoc:
   - https://hellodoc.app/s/27u6a/
   - https://hellodoc.app/s/gdgjq/
4. Клиент сам выбирает лабораторию/показатели в агрегаторе.
5. После получения результатов клиент возвращается в Telegram и присылает PDF/фото.
6. Бот формирует нутрициологический разбор.

### Если анализы уже есть

1. Клиент пишет боту.
2. Проходит анкету.
3. Присылает анализы в этот же Telegram-чат.
4. Бот собирает картину: жалобы, питание, образ жизни, анализы, гипотезы, красные флаги.
5. Клиент получает премиальный разбор.

## Что тарифицируется

### 1. Текстовая AI-модель

Используется для ответов в живом чате, формирования нутрициологических гипотез, составления Premium Wellness Dossier, проверки красных флагов и ответа на отзывы.

### 2. Распознавание голосовых

Используется только когда клиент отправляет голосовое или аудио.

### 3. Хостинг

Варианты:

- локальный компьютер: почти бесплатно, но ненадежно;
- VPS/облачная VM: стабильнее, платно каждый месяц;
- Yandex Serverless Containers: может быть дешевле при малой нагрузке, но потребует перевода бота на webhook-архитектуру.

## Хостинг

### Вариант 1. Локальный компьютер

Стоимость: 0 ₽ в месяц.

Плюсы:

- можно тестировать сразу;
- не нужен VPS;
- минимальные расходы.

Минусы:

- если компьютер выключен, бот не работает;
- если пропадает интернет, бот не работает;
- не production-ready.

Решение: подходит только для тестов и пилота на ограниченной аудитории.

### Вариант 2. VPS / облачная VM

Ориентир: примерно 700-1800 ₽ в месяц в зависимости от провайдера и конфигурации.

Для пилота хватит маленькой машины:

- 1-2 vCPU;
- 1-2 GB RAM;
- 10-20 GB disk.

Решение: оптимально для стабильного коммерческого запуска.

### Вариант 3. Yandex Serverless Containers

Тарифицируется по вызовам, CPU/RAM времени и трафику. Есть бесплатный лимит: первые 1 000 000 вызовов контейнера в месяц, 10 GB×hour RAM и 5 vCPU×hour CPU не тарифицируются.

Источник: Yandex Serverless Containers pricing.
https://yandex.cloud/ru/docs/serverless-containers/pricing

Важно: текущий бот работает polling-режимом. Для Serverless нужно переделать на webhook-архитектуру.

Решение: экономически хорошо, но технически это отдельный этап.

## Рекомендация архитектора

На сейчас не надо уходить в полностью Яндекс ради AI-модели.

Оптимальная схема:

1. DeepSeek V4 Flash для текстов и разборов.
2. Yandex SpeechKit для голосовых.
3. Локальный запуск для тестов.
4. После первых живых клиентов — перенос на VPS.
5. Serverless Containers рассматривать позже, когда будет понятно количество обращений и нужна ли webhook-архитектура.

## Коммерческий вывод

При цене продукта 2 900-9 900 ₽ себестоимость AI-инфраструктуры в гибридной схеме почти незаметна:

- при 2 900 ₽: около 0,13% от выручки на клиента;
- при 4 900 ₽: около 0,08%;
- при 9 900 ₽: около 0,04%.

Даже если реальный расход токенов вырастет в 5 раз, переменная AI-себестоимость гибридной схемы останется низкой.

Главные риски сейчас не стоимость API, а:

- качество юридических формулировок;
- отсутствие финальной оплаты ЮKassa;
- стабильный хостинг;
- корректная обработка анализов без медицинских назначений;
- тестирование живого сценария от первого сообщения до PDF.
